# 第三层：内容生成层（单项目）

**文档类型**: 内容生成规范
**适用范围**: 每个影视飓风风格视频项目的内容生成阶段
**前置条件**:
- 第一层（方法论与标准建立）已完成
- 第二层（项目准备层）已完成

---

## 一、层次定位

本层属于**内容生成层**，核心任务是基于第一层的风格分析和第二层的研究内容，通过6个核心模块逐步生成最终的内容输出物（叙事结构、配音文案、分镜脚本、素材清单）。

**特点**：
- 线性工作流：6个模块按顺序执行
- 每个模块有明确的输入输出
- 上下模块之间有强依赖关系
- 一次性生成，不涉及迭代优化

**与上下层的关系**：
```
第一层：方法论与标准建立
    ↓ 提供风格分析、结构模板
第二层：项目准备层
    ↓ 提供选题研究、原始信息量
第三层：内容生成层（本层）← 执行
    ↓ 产出最终内容物
第四层：Skill链沉淀
```

---

## 二、核心模块（6个）

### 模块1：逻辑主线生成

#### 目的
基于影视飓风风格分析和原始信息量，提炼出核心观点、反转点和情感曲线，作为后续内容生成的基础框架。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| 影视飓风内容形式与结构分析报告 | 第一层 | 风格参考、结构标准 |
| 分镜节奏与脚本分析 | 第一层 | 节奏参考、信息密度标准 |
| 原始信息量文档 | 第二层 | 选题研究内容、核心信息 |
| 需求包内容 | 第二层 | 选题、时长、约束条件 |

#### 提示词/操作指令

```
请基于以下内容生成逻辑主线文档：

【输入资料】
1. 影视飓风内容形式与结构分析报告：
   {附上分析报告内容或文件路径}

2. 分镜节奏与脚本分析：
   {附上分析内容或文件路径}

3. 原始信息量文档：
   {附上文档内容或文件路径}

4. 需求包内容：
   - 选题：{选题名称}
   - 预期时长：{如：12分钟}
   - 目标信息密度：{如：4-6点/分钟}
   - 约束条件：{如：无实拍、预算限制等}

【输出要求】
逻辑主线文档应包含以下部分：

1. 核心观点（5-7个）
   - 每个观点作为后续章节的基础
   - 观点之间要有逻辑递进关系
   - 观点要能支撑起影视飓风风格的信息密度

2. 反转点设计（至少2个）
   - 第一个反转点：打破观众常识认知（通常在中段）
   - 第二个反转点：价值升华或哲学反思（通常在结尾前）
   - 每个反转点要标注预期出现的时机和效果

3. 情感曲线设计
   - 开场：紧张/好奇（制造钩子）
   - 中段：理性/探索（技术解释）
   - 结尾：感动/震撼（价值升华）
   - 标注每个情感阶段的时长比例

4. 章节划分建议
   - 基于核心观点划分章节（5-7章）
   - 每章的时长分配
   - 每章的情绪基调和信息密度目标

【格式要求】
输出为Markdown格式，文件命名为"逻辑主线.md"
使用清晰的层级结构（##、###、-等）
```

#### 输出
- **文件**: `逻辑主线.md`
- **内容**:
  - 核心观点（5-7个）
  - 反转点设计（2个以上）
  - 情感曲线
  - 章节划分建议

#### 与下一模块的关联
- **输出内容** → 模块2（叙事结构）的输入之一
- **关联强度**: 强依赖（模块2必须基于本模块输出的逻辑主线）

### 模块2：叙事结构生成

#### 目的
基于逻辑主线，结合影视飓风风格标准和素材约束，生成详细的叙事结构文档，包含每个章节的具体内容、信息密度和视觉规划。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| 逻辑主线md文档 | 模块1输出 | 核心观点、反转点、情感曲线 |
| 需求包内容 | 第二层 | 选题、时长、约束条件 |
| 影视飓风内容与结构分析报告 | 第一层 | 风格参考、结构标准 |
| 分镜节奏与脚本分析 | 第一层 | 节奏参考、信息密度标准 |
| 附加提示词 | 用户输入 | 素材约束：只能来源于网络查找及AI生成，无法实拍 |

#### 提示词/操作指令

```
请基于以下内容生成叙事结构文档：

【输入资料】
1. 逻辑主线文档：
   {附上文档内容或文件路径}

2. 需求包内容：
   - 选题：{选题名称}
   - 预期时长：{如：12分钟}
   - 目标信息密度：{如：4-6点/分钟}

3. 影视飓风内容与结构分析报告：
   {附上分析报告内容或文件路径}

4. 分镜节奏与脚本分析：
   {附上分析内容或文件路径}

5. 附加约束条件：
   ⚠️ 素材只能来源于：网络查找（Pexels、Pixabay等免费素材网站）及AI生成（图片、视频），无法实拍

【输出要求】
叙事结构文档应包含以下部分：

1. 章节结构总览
   - 章节数量：{如：7章}
   - 总时长：{如：12分钟}
   - 每章的序号、标题、类型、时长

2. 每章详细设计（对每个章节）
   - 章节类型：开场钩子 / 历史叙事 / 技术解释 / 反转 / 升华
   - 情绪基调：紧张 / 理性 / 惊喜 / 感动
   - 信息密度：X.X 点/分钟
   - 主要视觉：信息图 X% / AI动画 X% / 实拍 X%
   - 叙事要点：
     * 开场：如何制造钩子？
     * 冲突：核心问题是什么？
     * 解释：如何用数据支撑？
     * 反转：反转点如何设计？
     * 升华：如何价值升华？
   - 可视化素材规划：
     * 需要哪些信息图？
     * 需要哪些AI生成内容？
     * 需要哪些网络素材？

3. 信息密度统计
   - 总信息点数量
   - 每分钟信息密度
   - 各章节信息密度分布

4. 视觉规划总览
   - 信息图清单（编号、标题、类型）
   - AI素材清单（图片、视频）
   - 网络素材清单（来源、关键词）

【格式要求】
输出为Markdown格式，文件命名为"叙事结构-v1.0.md"
使用表格、列表等结构化格式
确保所有素材规划符合"无实拍"约束
```

#### 输出
- **文件**: `叙事结构-v1.0.md`
- **内容**:
  - 章节结构总览
  - 每章详细设计（类型、情绪、密度、视觉、叙事要点、可视化素材）
  - 信息密度统计
  - 视觉规划总览

#### 与下一模块的关联
- **输出内容** → 模块3（配音文案）的主要输入
- **输出内容** → 模块4（分镜脚本brief）的输入之一
- **关联强度**: 强依赖（模块3、4必须基于本模块输出的叙事结构）

### 模块3：配音文案生成

#### 目的
基于叙事结构，生成完整的配音文案文本，包含所有章节的台词内容，确保与叙事结构的信息密度和节奏相匹配。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| 叙事结构md文档 | 模块2输出 | 章节结构、叙事要点、信息密度 |
| 第一层风格参考（隐式） | 第一层 | 影视飓风的文案风格（简洁、有力、技术感） |

#### 提示词/操作指令

```
请基于以下内容生成配音文案：

【输入资料】
1. 叙事结构文档：
   {附上文档内容或文件路径}

【输出要求】
配音文案应包含以下特点：

1. 风格要求
   - 简洁有力，避免冗余表达
   - 技术术语要有上下文解释
   - 使用设问、反问制造互动感
   - 数据要精确，对比要鲜明
   - 适度的停顿和重读标记

2. 结构要求
   - 按章节组织，每章有明确的标题分隔
   - 每章开头标注预计时长
   - 台词自然流畅，口语化但不失专业

3. 内容要求
   - 严格对应叙事结构中的叙事要点
   - 确保信息密度达标（4-6点/分钟）
   - 开场要有钩子（制造好奇）
   - 中段要有解释（数据支撑）
   - 结尾要有升华（价值提升）
   - 反转点要有明显标记

4. 格式要求
   - 使用Markdown格式
   - 每章用分隔线明确区分（══════）
   - 每章标注：【章节X: 标题】和预计时长
   - 台词分段清晰，便于配音时把握节奏
   - 可选：标注停顿（...）和重读（**加粗**）

【示例格式】
═══════════════════════════════════════
【章节1: 开场钩子】
预计时长：1分钟
═══════════════════════════════════════

开场钩子台词1...

台词2...

台词3...（此处制造悬念）

═══════════════════════════════════════
【章节2: 历史叙事】
预计时长：2分钟
═══════════════════════════════════════

...

【文件命名】
输出为"配音文案.txt"或"配音文案.md"
```

#### 输出
- **文件**: `配音文案.txt` 或 `配音文案.md`
- **内容**:
  - 完整的配音文案文本
  - 按章节组织，标注时长
  - 包含停顿和重读标记（可选）

#### 与下一模块的关联
- **输出内容** → 模块4（分镜脚本brief）的输入之一
- **输出内容** → 模块5（详细分镜脚本）的输入之一
- **关联强度**: 强依赖（模块4、5必须基于本模块输出的配音文案）

### 模块4：分镜脚本Brief生成

#### 目的
基于叙事结构和配音文案，生成分镜脚本brief清单，定义每个镜头的序号、概要、时长、素材来源和优先级，作为后续详细分镜制作的基础框架。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| 叙事结构md文档 | 模块2输出 | 章节结构、视觉规划 |
| 配音文案md/txt | 模块3输出 | 完整台词内容 |

#### 提示词/操作指令

```
请基于以下内容生成分镜脚本brief清单：

【输入资料】
1. 叙事结构文档：
   {附上文档内容或文件路径}

2. 配音文案：
   {附上文案内容或文件路径}

【输出要求】
分镜脚本brief清单应包含：

1. 文档元信息
   - 文档版本、创建日期
   - 基于文档（叙事结构版本、配音文案版本）
   - 总时长、总镜头数、风格对标
   - 使用说明、镜头编号规则、优先级标记

2. 章节结构（对每个章节）
   - 章节类型、情绪基调、信息密度、主要视觉
   - 镜头序列表格：
     * 镜头编号（格式：[章节号]-[镜头序号]，如：1-01）
     * 镜头概要（一句话描述）
     * 时长（秒）
     * 素材来源（网络素材#X / AI图片#X / AI视频#X / 信息图#X）
     * 优先级（⭐⭐⭐ / ⭐⭐ / ⭐）
     * 对应配音文案位置（标注台词位置）

3. 章节统计
   - 镜头数、总时长、转场位置

4. 全片统计
   - 总体数据（总镜头数、总时长、平均时长）
   - 按章节统计表格
   - 按素材类型统计表格

5. 附录
   - 素材编号对照表（网络素材、AI图片、AI视频、信息图）

6. 后续制作建议
   - 分镜脚本制作建议
   - 优先级制作顺序
   - 质量检查要点

【关键要求】
- 镜头划分要保证叙事连贯性
- 每个镜头时长合理（5-12秒）
- 素材来源与叙事结构中的视觉规划对应
- 优先级标记准确（⭐⭐⭐必须镜头、⭐⭐重要镜头、⭐可选镜头）
- 配音文案位置标注清晰

【文件命名】
输出为"镜头brief清单.md"
```

#### 输出
- **文件**: `镜头brief清单.md`
- **内容**:
  - 文档元信息和使用说明
  - 章节结构和镜头序列表格
  - 章节统计和全片统计
  - 素材编号对照表
  - 后续制作建议

#### 与下一模块的关联
- **输出内容** → 模块5（详细分镜脚本）的输入之一
- **关联强度**: 强依赖（模块5必须基于本模块输出的镜头brief框架）

### 模块5：详细分镜脚本生成（无头模式）

#### 目的
基于叙事结构、配音文案和镜头brief清单，使用无头模式逐个生成单个分镜镜头的详细元素，最后检查总体连贯性，产出完整的详细分镜脚本。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| 叙事结构md文档 | 模块2输出 | 章节结构、情绪基调、信息密度 |
| 配音文案md/txt | 模块3输出 | 完整台词内容 |
| 镜头brief清单md | 模块4输出 | 镜头拓扑、职责边界、素材来源 |

#### 提示词/操作指令（分两阶段）

##### 阶段1：无头模式逐镜头生成

```
claude -p 【进入无头模式分镜生成状态】

请以以下内容为基础逐个生成详细分镜：

【唯一镜头拓扑来源】
镜头brief清单.md：{附上文件路径或内容}
⚠️ 严格按照brief清单的镜头顺序和职责边界，不得新增、合并、重排镜头

【全局叙事约束】
叙事结构-v1.0.md：{附上文件路径或内容}
⚠️ 确保每个镜头的情绪基调、信息密度与叙事结构一致

【配音文本来源】
配音文案.txt：{附上文件路径或内容}
⚠️ 每个镜头的配音文案必须精确对应

【素材来源参考】
素材清单-方案B_平衡推荐.md：{附上文件路径或内容}
⚠️ 视觉元素只引用素材名称，不描述具体画面

【生成规则】
1. 严格按照brief清单顺序，逐个镜头生成
2. 每次只处理当前镜头，不重复已完成镜头的功能
3. 不提前覆盖后续镜头的职责
4. 每个镜头包含以下12个必需要素（不可省略）：
   - 基本信息
   - 叙事目的
   - 视觉元素（仅素材名称）
   - 镜头运动
   - 色彩要求
   - 细节要求
   - 配音文案
   - 配音要求
   - 背景音乐和音效
   - 转场设计
   - 技术参数
   - 注意事项

5. 视觉元素简化规则：
   ❌ 错误："键盘打字特写，手指敲击字母键，蓝色背光，科技感背景"
   ✅ 正确："键盘打字特写素材（网络素材#1，Pexels）"

6. 每个镜头生成后自动进行连贯性自检（不写入文档）：
   - 检查与前镜头的语义连续性
   - 检查与配音文案的对应准确性
   - 检查素材引用的正确性

【开始生成】
从镜头1-01开始，按顺序逐个生成，生成到镜头7-10为止。
```

##### 阶段2：总体连贯性检查

```
【总体连贯性检查】

请检查已生成的详细分镜脚本，重点关注：

1. 镜头间连贯性
   - 视觉转场是否流畅？
   - 配音是否有断层或跳跃？
   - 情绪基调转换是否自然？

2. 配音对应准确性
   - 每个镜头的配音文案是否与配音文案.txt完全一致？
   - 配音要求（语速、语气、停顿）是否合理？

3. 素材引用准确性
   - 每个镜头的素材引用是否与素材清单一致？
   - 素材编号是否正确？
   - 是否存在引用不存在的素材？

4. 完整性检查
   - 是否所有镜头都已生成（64个镜头）？
   - 每个镜头是否包含12个要素？
   - 是否存在要素缺失的情况？

5. 统计数据准确性
   - 总时长是否为490秒（8分10秒）？
   - 各章节镜头数和时长是否与brief清单一致？

【输出要求】
如果发现问题，列出具体位置（镜头编号、要素名称）和问题描述
如果全部通过，输出"✅ 连贯性检查通过，共X个镜头，Y秒，无问题"
```

#### 输出
- **文件**: `分镜脚本-详细版.md`
- **内容**:
  - 所有64个镜头的详细分镜
  - 每个镜头包含12个要素
  - 章节完成标记
  - 全片完成总结和统计

#### 与下一模块的关联
- **输出内容** → 模块6（素材清单）的输入
- **关联强度**: 强依赖（模块6必须基于本模块输出的详细分镜脚本）

### 模块6：素材清单生成

#### 目的
基于详细分镜脚本，整理并输出完整的素材清单，包含所有需要的素材来源、获取方式、预算和时间分配。

#### 输入

## 七、常见问题

### Q1: 如果某个模块的输出不满足要求怎么办？
- 本工作流假设一次生成即满足要求
- 如需修改，返回相应模块，调整提示词重新生成
- 不建议反复迭代，以免破坏模块间的对应关系

### Q2: 模块5的"无头模式"是什么意思？
- 无头模式指：严格按照brief清单逐个生成，不添加主观判断
- 每次只处理当前镜头，不预判后续内容
- 生成后自动进行连贯性自检（但不写入文档）

### Q3: 为什么模块6放在最后而不是模块2/3之后？
- 详细分镜脚本确定后，素材需求才最终明确
- 避免因脚本调整导致素材清单反复修改
- 确保素材清单与最终脚本完全对应

### Q4: 6个模块必须按顺序执行吗？
- 是的，必须按顺序执行
- 每个模块的输出是下一模块的输入
- 跳过模块会导致输入缺失，无法继续

### Q5: 如果想中途调整前面的内容怎么办？
- 可以返回相应模块重新生成
- 但要注意：修改某模块后，后续所有模块都要重新生成
- 确保所有文档的版本号对应（如：v1.0、v1.1）

## 八、执行注意事项

### 关键原则
1. **严格顺序执行**：6个模块必须按1→2→3→4→5→6顺序执行
2. **保持版本一致**：所有文档应标注相同的版本号（如：v1.0）
3. **输入输出明确**：每个模块开始前确认所有输入都已准备
4. **质量检查及时**：每个模块完成后立即进行质量检查
5. **文档规范命名**：使用统一的命名规范，便于追溯和管理

### 提示词优化建议
- 在每个模块的提示词中，尽量引用文件路径而非内容，避免提示词过长
- 根据实际情况调整约束条件（如预算、时长、素材来源）
- 保留历史版本的提示词，便于对比和优化

### 输出物管理
- 所有输出物统一放在项目目录下
- 建议目录结构：
  ```
  项目名称/
  ├── 逻辑主线.md
  ├── 叙事结构-v1.0.md
  ├── 配音文案.txt
  ├── 镜头brief清单.md
  ```
