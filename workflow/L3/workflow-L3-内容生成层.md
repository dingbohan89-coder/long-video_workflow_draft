# 第三层：内容生成层（单项目）

**文档类型**: 内容生成规范
**适用范围**: 每个影视飓风风格视频项目的内容生成阶段
**前置条件**:
- 第一层（方法论与标准建立）已完成
- 第二层（准备层）已完成

---

## 一、层次定位

本层属于**内容生成层**，核心任务是基于第一层的风格分析和第二层的研究内容，通过6个核心模块逐步生成最终的内容输出物（叙事结构、配音文案、分镜脚本、素材清单）。

**特点**：
- 线性工作流：6个模块按顺序执行
- 每个模块有明确的输入输出
- 上下模块之间有强依赖关系
- 一次性生成，不涉及迭代优化

**与上下层的关系**：
```
第一层：影视飓风长视频风格调研
    ↓ 提供风格分析、结构模板
第二层：选题调研分析
    ↓ 提供选题研究、原始信息量
第三层：内容生成层（本层）← 执行
    ↓ 产出最终内容物
第四层：Skill链沉淀
```

---

## 二、核心模块（6个）

### 模块1：逻辑主线生成

#### 目的
基于影视飓风风格分析和原始信息量，提炼出核心观点、反转点和情感曲线，作为后续内容生成的基础框架。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| 影视飓风内容形式与结构分析报告 | 第一层 | 风格参考、结构标准 |
| 分镜节奏与脚本分析 | 第一层 | 节奏参考、信息密度标准 |
| 原始信息量文档 | 第二层 | 选题研究内容、核心信息 |
| 需求包内容 | 第二层 | 选题、时长、约束条件 |

#### 提示词/操作指令

```
请基于以下内容生成逻辑主线文档：

【输入资料】
1. 影视飓风内容形式与结构分析报告：
   {附上分析报告内容或文件路径}

2. 分镜节奏与脚本分析：
   {附上分析内容或文件路径}

3. 原始信息量文档：
   {附上文档内容或文件路径}

4. 需求包内容：
   - 选题：{选题名称}
   - 预期时长：{如：12分钟}
   - 目标信息密度：{如：4-6点/分钟}
   - 约束条件：{如：无实拍、预算限制等}

【输出要求】
逻辑主线文档应包含以下部分：

1. 核心观点（5-7个）
   - 每个观点作为后续章节的基础
   - 观点之间要有逻辑递进关系
   - 观点要能支撑起影视飓风风格的信息密度

2. 反转点设计（至少2个）
   - 第一个反转点：打破观众常识认知（通常在中段）
   - 第二个反转点：价值升华或哲学反思（通常在结尾前）
   - 每个反转点要标注预期出现的时机和效果

3. 情感曲线设计
   - 开场：紧张/好奇（制造钩子）
   - 中段：理性/探索（技术解释）
   - 结尾：感动/震撼（价值升华）
   - 标注每个情感阶段的时长比例

4. 章节划分建议
   - 基于核心观点划分章节（5-7章）
   - 每章的时长分配
   - 每章的情绪基调和信息密度目标

【格式要求】
输出为Markdown格式，文件命名为"逻辑主线.md"
使用清晰的层级结构（##、###、-等）
```

#### 输出
- **文件**: `逻辑主线.md`
- **内容**:
  - 核心观点（5-7个）
  - 反转点设计（2个以上）
  - 情感曲线
  - 章节划分建议

#### 与下一模块的关联
- **输出内容** → 模块2（叙事结构）的输入之一
- **关联强度**: 强依赖（模块2必须基于本模块输出的逻辑主线）

### 模块2：叙事结构生成

#### 目的
基于逻辑主线，结合影视飓风风格标准和素材约束，生成详细的叙事结构文档，包含每个章节的具体内容、信息密度和视觉规划。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| 逻辑主线md文档 | 模块1输出 | 核心观点、反转点、情感曲线 |
| 需求包内容 | 第二层 | 选题、时长、约束条件 |
| 影视飓风内容与结构分析报告 | 第一层 | 风格参考、结构标准 |
| 分镜节奏与脚本分析 | 第一层 | 节奏参考、信息密度标准 |
| 附加提示词 | 用户输入 | 素材约束：只能来源于网络查找及AI生成，无法实拍 |

#### 提示词/操作指令

```
请基于以下内容生成叙事结构文档：

【输入资料】
1. 逻辑主线文档：
   {附上文档内容或文件路径}

2. 需求包内容：
   - 选题：{选题名称}
   - 预期时长：{如：12分钟}
   - 目标信息密度：{如：4-6点/分钟}

3. 影视飓风内容与结构分析报告：
   {附上分析报告内容或文件路径}

4. 分镜节奏与脚本分析：
   {附上分析内容或文件路径}

5. 附加约束条件：
   ⚠️ 素材只能来源于：网络查找（Pexels、Pixabay等免费素材网站）及AI生成（图片、视频），无法实拍

【输出要求】
叙事结构文档应包含以下部分：

1. 章节结构总览
   - 章节数量：{如：7章}
   - 总时长：{如：12分钟}
   - 每章的序号、标题、类型、时长

2. 每章详细设计（对每个章节）
   - 章节类型：开场钩子 / 历史叙事 / 技术解释 / 反转 / 升华
   - 情绪基调：紧张 / 理性 / 惊喜 / 感动
   - 信息密度：X.X 点/分钟
   - 主要视觉：信息图 X% / AI动画 X% / 实拍 X%
   - 叙事要点：
     * 开场：如何制造钩子？
     * 冲突：核心问题是什么？
     * 解释：如何用数据支撑？
     * 反转：反转点如何设计？
     * 升华：如何价值升华？
   - 可视化素材规划：
     * 需要哪些信息图？
     * 需要哪些AI生成内容？
     * 需要哪些网络素材？

3. 信息密度统计
   - 总信息点数量
   - 每分钟信息密度
   - 各章节信息密度分布

4. 视觉规划总览
   - 信息图清单（编号、标题、类型）
   - AI素材清单（图片、视频）
   - 网络素材清单（来源、关键词）

【格式要求】
输出为Markdown格式，文件命名为"叙事结构-v1.0.md"
使用表格、列表等结构化格式
确保所有素材规划符合"无实拍"约束
```

#### 输出
- **文件**: `叙事结构-v1.0.md`
- **内容**:
  - 章节结构总览
  - 每章详细设计（类型、情绪、密度、视觉、叙事要点、可视化素材）
  - 信息密度统计
  - 视觉规划总览

#### 与下一模块的关联
- **输出内容** → 模块3（配音文案）的主要输入
- **输出内容** → 模块4（分镜脚本brief）的输入之一
- **关联强度**: 强依赖（模块3、4必须基于本模块输出的叙事结构）

### 模块3：配音文案生成

#### 目的
基于叙事结构，生成完整的配音文案文本，包含所有章节的台词内容，确保与叙事结构的信息密度和节奏相匹配。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| 叙事结构md文档 | 模块2输出 | 章节结构、叙事要点、信息密度 |
| 第一层风格参考（隐式） | 第一层 | 影视飓风的文案风格（简洁、有力、技术感） |

#### 提示词/操作指令

```
请基于以下内容生成配音文案：

【输入资料】
1. 叙事结构文档：
   {附上文档内容或文件路径}

【输出要求】
配音文案应包含以下特点：

1. 风格要求
   - 简洁有力，避免冗余表达
   - 技术术语要有上下文解释
   - 使用设问、反问制造互动感
   - 数据要精确，对比要鲜明
   - 适度的停顿和重读标记

2. 结构要求
   - 按章节组织，每章有明确的标题分隔
   - 每章开头标注预计时长
   - 台词自然流畅，口语化但不失专业

3. 内容要求
   - 严格对应叙事结构中的叙事要点
   - 确保信息密度达标（4-6点/分钟）
   - 开场要有钩子（制造好奇）
   - 中段要有解释（数据支撑）
   - 结尾要有升华（价值提升）
   - 反转点要有明显标记

4. 格式要求
   - 使用Markdown格式
   - 每章用分隔线明确区分（══════）
   - 每章标注：【章节X: 标题】和预计时长
   - 台词分段清晰，便于配音时把握节奏
   - 可选：标注停顿（...）和重读（**加粗**）

【示例格式】
═══════════════════════════════════════
【章节1: 开场钩子】
预计时长：1分钟
═══════════════════════════════════════

开场钩子台词1...

台词2...

台词3...（此处制造悬念）

═══════════════════════════════════════
【章节2: 历史叙事】
预计时长：2分钟
═══════════════════════════════════════

...

【文件命名】
输出为"配音文案.txt"或"配音文案.md"
```

#### 输出
- **文件**: `配音文案.txt` 或 `配音文案.md`
- **内容**:
  - 完整的配音文案文本
  - 按章节组织，标注时长
  - 包含停顿和重读标记（可选）

#### 与下一模块的关联
- **输出内容** → 模块4（分镜脚本brief）的输入之一
- **输出内容** → 模块5（详细分镜脚本）的输入之一
- **关联强度**: 强依赖（模块4、5必须基于本模块输出的配音文案）

### 模块4：分镜脚本Brief生成

#### 目的
基于SRT字幕文件和影视飓风分镜节奏参考，生成分镜脚本Brief清单，规范每个分镜镜头的时长并解决配音文案的连贯性问题，同时基于混合素材策略进行素材类型分配和复杂镜头拆分。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| SRT字幕文件 | 模块3输出转换 | 配音内容+时间码信息 |
| 影视飓风分镜节奏参考文档 | 第一层 | 节奏标准、信息密度、分镜范式 |
| 素材类型约束 | 用户定义 | 4种素材类型定义 |
| Brief维度约束 | 用户定义 | 5个维度要求 |

#### 提示词/操作指令

```
请基于以下内容生成分镜脚本Brief清单：

【输入资料】
1. SRT字幕文件：
   {附上SRT文件内容或文件路径}

2. 影视飓风分镜节奏参考文档：
   {附上文档内容或文件路径}

3. 素材类型约束（4种）：
   - Figma演示：品牌logo/数据/对比/技术原理/交互演示
   - 图表图片：独立统计图表（柱状图、饼图等）
   - AI单逻辑素材：纯情绪/氛围渲染，可裁剪（代码雨、神经网络、粒子效果等）
   - AI图片：具象化场景、比喻场景

4. Brief维度约束（5个维度）：
   - 时间码
   - 镜头时长
   - 配音文案
   - 素材类型
   - 设计思路

【输出要求】
分镜脚本Brief清单应包含：

1. 文档元信息
   - 文档版本、创建日期
   - 基于文档（SRT字幕版本、参考文档版本）
   - 总时长、总镜头数、风格对标
   - 使用说明、镜头编号规则

2. 拆解分镜的工作范式（5个原则）
   - 逻辑完整性原则：逻辑关联的句子合并为一个镜头（对比、因果、递进关系不要拆散）
   - 素材能力最大化原则：Figma用于对比/数据/交互，AI动画用于抽象氛围
   - 视觉密度原则：一个镜头承载一个完整的信息单元
   - 节奏变化：关键信息3-5秒，过渡2-3秒，形成"密-疏-密"节奏
   - 参考影视飓风风格：开篇Hook极致对比、数据展示并排对比、审美留白

3. 素材类型分配规则
   - 优先选择Figma演示：品牌logo/UI界面/精确文字/分屏对比/数据可视化
   - 优先选择AI单逻辑素材：抽象氛围/情绪化场景/自然概念画面
   - 复杂镜头拆分策略：可拆分成多个部分（数据部分用Figma，情绪部分用AI素材）

4. 章节结构（对每个章节）
   - 镜头序列表格：
     * 镜头编号（格式：[章节号]-[镜头序号]，如：1-01）
     * 时间码（起始时间-结束时间）
     * 镜头时长（秒）
     * 配音文案（对应SRT字幕内容）
     * 素材类型（Figma演示/图表图片/AI单逻辑素材/AI图片）
     * 设计思路（为什么选择这种素材类型，如何拆分）

5. 全片统计
   - 总体数据（总镜头数、总时长、平均时长）
   - 按素材类型统计表格

6. AI单逻辑素材说明
   - 定义：循环重复、无叙事逻辑、任意时段截取效果相同
   - 示例：代码雨、神经网络、粒子旋转、光束流动、数据流、代码滚动、敲代码演示、读书演示

【关键要求】
- 基于SRT时间码规范每个镜头的时长
- 保证配音文案的连贯性（不要在逻辑关联处断开）
- 素材类型分配符合混合素材策略
- 复杂镜头可以拆分成多个素材组合
- 设计思路要清晰说明素材选择的理由

【文件命名】
输出为"镜头brief清单.md"
```

#### 输出
- **文件**: `镜头brief清单.md`
- **内容**:
  - 文档元信息和使用说明
  - 拆解分镜的工作范式和素材类型分配规则
  - 章节结构和镜头序列表格（5个维度）
  - 全片统计和AI单逻辑素材说明

#### 与下一模块的关联

### 模块5：详细分镜脚本生成

#### 目的
基于镜头Brief清单，逐个生成单个分镜镜头的详细元素（6个维度），每个分镜创建独立的md文档，最后组装成完整分镜脚本。

#### 输入

| 输入物 | 来源 | 说明 |
|--------|------|------|
| 镜头Brief清单md | 模块4输出 | 镜头框架、素材类型分配、设计思路 |
| 分镜维度约束 | 用户定义 | 6个维度要求 |
| 素材类型约束 | 用户定义 | 4种素材类型定义 |

#### 提示词/操作指令（分三阶段）

##### 阶段1：逐个生成分镜镜头md文档

```
请基于以下内容逐个生成分镜镜头详细文档：

【唯一镜头框架来源】
镜头brief清单.md：{附上文件路径或内容}
⚠️ 严格按照Brief清单的镜头顺序，逐个生成

【分镜维度约束】（6个维度）
1. 配音文案：对应Brief中的配音文案内容
2. 镜头时长：对应Brief中的时长
3. 素材类型：对应Brief中的素材类型（4种）
4. 视觉画面：根据设计思路具体描述视觉内容
5. 镜头运动：基于配音分析规则设计（见下方）
6. BGM背景音乐和背景音效：配合情绪基调

【素材类型约束】（4种）
- Figma演示：品牌logo/数据/对比/技术原理/交互演示
- 图表图片：独立统计图表（柱状图、饼图等）
- AI单逻辑素材：纯情绪/氛围渲染，可裁剪
- AI图片：具象化场景、比喻场景

【镜头运动设计规则】（基于配音分析）
1. 配音分析规则：
   - 标记配音文案中的强调词（最高级、对比词、关键数字、情绪词）
   - 确定强调词出现的时间点（在前70%？后30%？最后5秒？）

2. 镜头运动匹配规则：
   - 如果强调词在前70%时间段 → 聚焦强调 → 后期拉远全景
   - 如果强调词在后30%时间段 → 聚焦强调到底 → 不拉远或轻微调整
   - 如果强调词在最后5秒 → 持续聚焦强调 → 直接硬切到下一个分镜
   - 如果没有明确强调词 → 可以在结尾拉远全景总结

3. 验证问题：
   对每个分镜的最后5秒，问自己："这个时候镜头是在强调重点，还是在削弱重点？"
   - 如果在削弱重点 → 修改镜头运动
   - 如果在强调重点 → 设计正确

4. 节奏优先级：
   配音重点 > 镜头运动美学
   宁可镜头运动简单，也不能削弱配音的情绪表达

【生成规则】
1. 严格按照Brief清单顺序，逐个镜头生成
2. 每个镜头创建一个独立的md文档，按镜头编号命名（如：1-01.md）
3. 每个镜头文档包含6个维度，不包含其他冗余信息
4. 视觉画面描述要具体但简洁（不超过50字）
5. 镜头运动要标注明确的运动类型（推/拉/摇/移/跟/升降/静止）
6. BGM和音效要具体到音乐类型和音效名称

【开始生成】
```

## 七、常见问题

### Q1: 如果某个模块的输出不满足要求怎么办？
- 本工作流假设一次生成即满足要求

### Q2: 模块4为什么要基于SRT字幕文件而不是配音文案txt？
- SRT字幕文件包含时间码信息，可以精确控制每个镜头的时长
- 解决配音文案连贯性问题（基于时间码判断逻辑断点）
- 规范分镜镜头的时长，避免过长或过短

### Q3: 什么是"混合素材策略"？
- 核心问题：AI视频工具不支持自定义时长，只能生成5-10秒
- 解决策略：需要精确时长的素材改为Figma演示动画，纯情绪素材保留为AI单逻辑素材（可裁剪）
- 复杂镜头可以拆分：数据部分(Figma) + 情绪部分(AI素材)

### Q4: 什么是"AI单逻辑素材"？
- 定义：循环重复、无叙事逻辑、任意时段截取效果相同
- 示例：代码雨、神经网络、粒子旋转、光束流动、数据流、代码滚动
- 优势：可以裁剪到任意时长，解决AI工具时长限制问题

### Q5: 模块5的"6个维度"是什么？
1. 配音文案
2. 镜头时长
3. 素材类型（4种）
4. 视觉画面
5. 镜头运动（基于配音分析规则）
6. BGM背景音乐和背景音效

### Q6: 模块6为什么要明确素材时长？
- 因为AI视频工具不支持自定义时长，素材时长可能与分镜时长对应不上
- Figma演示动画：标注精确时长，直接匹配分镜时长
- AI单逻辑素材：标注"可裁剪"，可以裁剪到目标时长
- 避免人工剪辑调整时长的工作

### Q7: 6个模块必须按顺序执行吗？
- 是的，必须按顺序执行

### Q8: 如果想中途调整前面的内容怎么办？
