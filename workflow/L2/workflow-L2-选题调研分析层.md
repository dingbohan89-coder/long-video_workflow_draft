## 一、层次定位

本层属于**项目准备层**，核心任务是理解需求、深度研究选题、建立知识护城河。

**特点**：
- 每个项目开始时执行一次
- 为具体项目定制化研究
- 产出项目专属的研究内容

**与上下层的关系**：
```
影视飓风长视频风格调研层（第一层）
    ↓ 提供工程化标准模板
选题调研分析（本层）← 使用
    ↓ 产出选题研究内容
内容生成层（第三层）
```

## 二、核心模块

### 模块1：深度研究（Deep Research）

#### 目的
建立知识护城河，确保内容专业性和数据支撑。

#### 我输入的内容
- 研究提示词
- 选题相关的基础文档
- 选题内容

#### 使用的工具
- Gemini Deep Research

#### 执行步骤

1. **打开Gemini**
   - 访问 gemini.google.com

2. **上传基础文档**（如有）
   - 上传老师或自己提供的相关文档

3. **输入研究提示词**

**示例：Markdown选题研究提示词**
   ```
   任务: 深度研究选题，为内容生成提供知识支撑

   研究主题:
   1. Markdown 历史
      - 2004年 John Gruber 与 Aaron Swartz 创始故事
      - 从技术圈到 AI 圈的演进路径
      - 关键节点：GitHub/StackOverflow adoption

   2. 技术对比（硬核数据）
      - Token 效率对比：Word vs PDF vs Markdown（需实测数据）
      - 文件体积对比：同样文章的 KB 数
      - RAG 系统检索准确率对比（引用研究论文）

   3. AI 原理深挖
      - LLM 如何解析 Markdown 结构
      - 为什么代码块语法是"人机桥梁"
      - 实际案例：ChatGPT/Claude 默认输出格式分析

   4. Obsidian 生态
      - 本地优先 vs 云端隐私风险对比
      - 双向链接的知识图谱可视化案例
      - AI 插件实际使用演示录屏

   5. 社会意义
      - 信息主权与格式选择
      - 开源标准 vs 商业格式的历史教训
      - AI 时代知识管理的范式转移

   执行方式:
   - 使用 Deep Research Agent 并行研究5个主题
   - 收集第一手资料：论文原文、GitHub commits、创始访谈
   - 生成 原始信息量文档.md（带引用来源）
   - 确保把选题在AI时代的重要度、原因、历史说明清晰
   ```

4. **整理输出**
   - 生成结构化的研究文档
   - 标注所有数据来源

#### 输出
- `原始信息量文档.md`

#### 关键研究重点
- 选题历史（诞生故事、发展节点）
- 技术对比（含实测数据）
- AI原理深挖（机制、优势）
- 生态和社会意义（开放标准、数据主权）

#### 质量标准
- [ ] 所有事实都有来源标注（URL或文献名）
- [ ] 至少3组对比数据
- [ ] 技术解释有类比（不是纯技术术语）
- [ ] 覆盖所有研究维度
- [ ] 数据来源可追溯

## 五、常见问题

### Q1: 深度研究必须做吗？
**A**: 必须做。这是建立知识护城河的关键，没有数据支撑的内容会显得空洞。

### Q2: 深度研究需要多长时间？
**A**: 约2-3小时，取决于选题复杂度。

### Q3: 如果找不到某些数据怎么办？
**A**: 明确标注"需补充实测"或"暂未找到"，不要编造数据。

### Q4: 为什么用Gemini做Deep Research？
**A**:
- Gemini 1.5 Pro支持长文本，可上传多个文档
- 擅长深度研究和综合分析
- 免费且效果好
