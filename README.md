# 影视飓风风格长视频制作工作流简介

**适用场景**：制作影视飓风（MediaStorm）风格的长视频（10-15分钟）
**核心理念**：工业化、标准化、模块化的视频生产流程
**总耗时**：3-5天（从选题到成片）

---

## 一、工作流三层架构

```
┌─────────────────────────────────────────────────────────────┐
│                    第一层：方法论与标准建立                    │
│                  （建立风格认知，制定制作标准）                  │
└─────────────────────────────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────┐
│                    第二层：项目准备层                          │
│               （选题研究、素材调研、需求定义）                   │
└─────────────────────────────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────┐
│                   第三层：内容生成层 ⭐核心                      │
│              （逻辑主线→叙事结构→配音文案→分镜→素材）             │
└─────────────────────────────────────────────────────────────┘
```

**简要说明**：
- **第一层**：只做一次，建立影视飓风风格的方法论和标准模板
- **第二层**：每个项目都要做，但内容因项目而异
- **第三层**：每个项目的核心执行环节，6个模块按顺序执行

---

## 二、核心概念解释

### 1. 叙事结构
**是什么**：整部视频的"骨架"，规划7个章节的内容和节奏
**作用**：
- 确定每个章节讲什么（开场、历史、技术解释、反转、升华）
- 控制信息密度（4-6个信息点/分钟）
- 规划视觉类型（信息图占多少、AI动画占多少）

**比喻**：就像建房子前先画蓝图，叙事结构就是视频的"蓝图"

### 2. 逻辑主线
**是什么**：整部视频的"核心观点"和"反转点"
**作用**：
- 提炼5-7个核心观点
- 设计2个反转点（打破常识认知、价值升华）
- 规划情绪曲线（紧张→理性→感动）

**比喻**：就像写文章前先列提纲，逻辑主线就是视频的"论点提纲"

### 3. 分镜脚本
**是什么**：每个镜头的详细执行方案
**作用**：
- 规范每个镜头的时长（如：镜头1-01，5秒）
- 明确素材类型（Figma演示/AI单逻辑素材/图表图片/AI图片）
- 设计镜头运动（推镜头/拉镜头/静止）
- 标注配音文案和BGM

**比喻**：就像电影的分镜头脚本，但更注重"素材类型分配"

## 三、内容生成层（L3）6个模块详解

```
模块1 → 模块2 → 模块3 → 模块4 → 模块5 → 模块6
  ↓      ↓      ↓      ↓      ↓      ↓
逻辑   叙事   配音   分镜   详细   素材
主线   结构   文案   Brief  脚本   提示词
```

### 模块1：逻辑主线生成
**输入**：影视飓风风格分析 + 选题研究内容
**输出**：`逻辑主线.md`
**做什么**：
- 提炼5-7个核心观点
- 设计2个反转点
- 规划情绪曲线

**举例**：
> 核心观点："Markdown是AI时代的人机协作桥梁"
> 反转点1：从技术效率→数据主权（Obsidian本地存储）
> 反转点2：从数据主权→格式即自由（开放标准的重要性）

---

### 模块2：叙事结构生成
**输入**：逻辑主线 + 预期时长（如12分钟）
**输出**：`叙事结构-v1.0.md`
**做什么**：
- 规划7个章节的结构
- 每章安排：内容要点、信息密度、视觉类型
- 确保总时长和节奏符合要求

**举例**：
```
章节1: 开场钩子（0:00-1:00）
- 情绪：紧张、好奇
- 信息密度：5.0点/分钟
- 视觉：信息图70%、AI动画30%

章节2: 历史的悖论（1:00-3:00）
- 情绪：理性
- 信息密度：4.0点/分钟
- 视觉：信息图50%、AI动画50%
...
```

---

### 模块3：配音文案生成
**输入**：叙事结构
**输出**：`配音文案.txt` + `配音文案.srt`
**做什么**：
- 撰写所有台词（约3000-5000字）
- 风格：简洁、有力、技术感
- 转换为SRT格式（带时间码）

**举例**：
> "你是否知道？当你把一份写有同样内容的Word文档和Markdown文档发给AI处理时，Word消耗了4282个Token，而Markdown只要1203个。差了整整3.5倍！"

**关键**：SRT字幕文件包含时间码，用于后续精确控制每个镜头的时长

---

### 模块4：分镜脚本Brief生成 ⭐关键
**输入**：SRT字幕文件 + 影视飓风分镜节奏参考 + 混合素材策略
**输出**：`镜头brief清单.md`
**做什么**：
- 基于SRT时间码拆分镜头
- 规范每个镜头的时长
- 分配素材类型（Figma/AI单逻辑素材/图表/AI图片）
- 复杂镜头拆分（数据+情绪）

**举例**：
```
镜头1-01：
- 时间码：00:00:00,000 → 00:00:05,000
- 时长：5秒
- 配音文案："你是否知道？当你把一份写有同样内容的Word..."
- 素材类型：Figma演示动画
- 设计思路：需要展示Word vs Markdown的Token对比，精确数据展示用Figma
```

**混合素材策略**：
- 需要精确时长+具体数据 → **Figma演示动画**
- 纯情绪/氛围渲染 → **AI单逻辑素材**（可裁剪）
- 独立统计图表 → **图表图片**
- 具象化场景比喻 → **AI图片**

**AI单逻辑素材**：
- 定义：循环重复、无叙事逻辑、任意时段效果相同
- 示例：代码雨、神经网络、粒子效果、光束流动
- 优势：可以裁剪到任意时长，解决AI工具时长限制问题

---

### 模块5：详细分镜脚本生成
**输入**：镜头brief清单.md
**输出**：单个分镜md文档（1-01.md、1-02.md...）+ `分镜脚本-完整版.md`
**做什么**：
- 为每个镜头补充6个维度的详细信息
- 设计镜头运动（基于配音分析规则）
- 组装成完整分镜脚本

**6个维度**：
1. **配音文案**：对应Brief中的文案
2. **镜头时长**：对应Brief中的时长
3. **素材类型**：Figma演示/图表图片/AI单逻辑素材/AI图片
4. **视觉画面**：具体描述（不超过50字）
5. **镜头运动**：推/拉/摇/移/跟/升降/静止（基于配音重点）
6. **BGM和音效**：音乐类型+音效名称

**举例**：
```
【镜头1-01】
1. 配音文案：你是否知道？当你把一份写有同样内容的Word文档和Markdown文档...
2. 镜头时长：5秒
3. 素材类型：Figma演示动画
4. 视觉画面：左侧Word Token计数器疯狂跳动，右侧Markdown平稳流动
5. 镜头运动：从全景推进到左侧Word计数器（强调"4282个"），快速切换到右侧（对比"1203个"）
6. BGM和音效：紧张电子音乐背景 + Token跳动音效
```

**镜头运动设计规则**：
- 分析配音中的强调词（如："3.5倍！"）
- 强调词在前70% → 聚焦强调后拉远全景
- 强调词在后30% → 聚焦强调到底
- 强调词在最后5秒 → 持续聚焦强调，直接硬切
- **核心原则**：配音重点 > 镜头运动美学

---

### 模块6：素材提示词生成
**输入**：单个分镜镜头md文档
**输出**：单个素材提示词md文档 + `素材提示词清单.md`
**做什么**：
- 为每个分镜生成对应的素材提示词
- 明确素材时长（与分镜对应）
- 将镜头运动写入Figma动画提示词
- 组装成完整素材清单

**举例**：
```
【素材-1-01】
- 素材类型：Figma演示动画
- 对应镜头：1-01
- 时长要求：5秒（与分镜镜头时长严格对应）
- 镜头运动：从全景推进到左侧Word计数器（展示4282），快速切换到右侧Markdown（展示1203）
- 提示词/制作说明：
  * 创建分屏对比布局（左Word vs 右Markdown）
  * 左侧：Token计数器从0快速跳动到4282，红色高亮
  * 右侧：Token计数器平稳滚动到1203，绿色高亮
  * 镜头运动：初始视角为全景，2秒时推进到左侧计数器，3.5秒时快速切换到右侧
  * 动画时长：5秒
  * 导出格式：MP4，1920x1080，60fps
- 优先级：⭐⭐⭐（必须镜头）
```

**素材提示词优化**：
- Figma演示动画：标注精确时长+镜头运动，直接匹配分镜
- AI单逻辑素材：标注"可裁剪"，可以裁剪到目标时长
- 图表图片/AI图片：不涉及时长（静态）

**验证问题**：
> "如果AI生成的视频时长不符合分镜要求，这个素材还能用吗？"
> - 不能 → 改为Figma演示或拆分成多个素材
> - 能（单逻辑素材可裁剪）→ 设计正确

## 四、工作流核心优势

### 1. 解决AI视频工具的时长限制问题
**问题**：AI视频工具（如Pika、Runway）不支持自定义时长，只能生成5-10秒
**解决**：
- 精确时长的素材 → 用Figma演示动画（支持精确时长）
- 纯情绪素材 → 用AI单逻辑素材（可裁剪到任意时长）

### 2. 保证配音文案的连贯性
**方法**：基于SRT字幕文件的时间码判断逻辑断点
**效果**：不会在逻辑关联处断开（对比、因果、递进关系不拆散）

### 3. 镜头运动服务于配音重点
**原则**：配音重点 > 镜头运动美学
**方法**：分析配音中的强调词位置，设计对应的镜头运动
**效果**：镜头运动不会削弱配音的情绪表达

### 4. 素材类型分配科学化
**规则**：品牌/data/UI → Figma；纯氛围 → AI单逻辑素材
**效果**：避免"需要精确数据却用AI视频"导致的返工

## 八、常见问题FAQ

**Q1: 为什么要先写配音文案再做分镜？**
A: 配音文案是视频的"时间轴基准"。有了文案和时间码，才能准确计算每个镜头的时长，避免过长或过短。

**Q2: 什么是"混合素材策略"？**
A: 简单说就是"精确的用Figma，氛围的用AI"。因为AI视频工具时长不可控，所以需要把需要精确时长的素材改成Figma演示动画。

**Q3: AI单逻辑素材为什么"可裁剪"？**
A: 因为它是循环重复的，没有叙事逻辑。比如"代码雨"，你截取前3秒、后3秒、中间3秒，效果都一样。

**Q4: 镜头运动为什么要"基于配音分析"？**
A: 为了配合配音的重点。比如配音强调"3.5倍"，镜头就应该聚焦到"3.5"这个数字上，而不是在最后拉远削弱重点。

**Q5: 6个模块可以跳过吗？**
A: 不可以。每个模块的输出是下一模块的输入，跳过会导致信息缺失。

**Q6: 整个工作流的核心是什么？**
A: **标准化**。通过标准化流程，让视频制作像"搭积木"一样，每个模块都有明确的输入输出，减少返工和试错。
